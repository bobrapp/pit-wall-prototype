Locked Decisions
Do not relitigate without sponsor signoff (Tariq Osman)
#DecisionRationaleSource
L-1 Second Opinion is a dedicated service, not a config variant of the primary forecast service Blast-radius isolation; independent eval lifecycle; separate budget tracking prevents cost pollution Eng Spec stub
L-2 Different LLM provider for Second Opinion than the primary forecast service Two divergence levers (framing + provider) produce meaningful disagreement. One alone is insufficient for reliable bear-case separation Design doc § prompt pack
L-3 Auto-trigger blocks Gate 3 commit when MDS ≥ 0.30 Material disagreement on plan-of-record is exactly when human signoff matters. This is a hard gate (PRD FR-SO-7). GOT mechanism provides the escape valve. PRD FR-SO-7
L-4 Cost cap is per-seat-daily, not per-org Aligns budget pressure with the seat invoking; prevents one user's over-invocation from starving others (CC-7) PRD CC-7
L-5 Replay determinism ≥ 98% is non-negotiable Auditability requirement from PRD. Without replay fidelity the HIBT ledger loses GovOps trust. Eval gate SO-5 enforces this. Eval gate SO-5
L-6 Service failure NEVER blocks the primary forecasting path Second Opinion is value-add, not gate-keeper. Graceful degradation is non-negotiable. GOT-03 handles LLM provider outage explicitly. Design doc § sequence diagrams

Resolved During Design
Closed before implementation began — rationale captured here for context
#QuestionResolution
R-1 Bear-case vs. bull-case vs. bottom-up framing for v1.0? Bear-case (FRAMING [C]). Goal is to surface disagreements that primary missed; bear-case bias maximizes that surface area at v1.0.
R-2 How many divergent factors to include in SME routing payload? All factors with weight > 0.1, max 5. Top factor becomes routing key.
R-3 Async-only or sync fast-path? Async-only for v1.0. Sync 200 path reserved for future cache-hit optimization.
R-4 Dismiss reason codes — how many? Three: DISM-1 (spurious signal), DISM-2 (stale data), DISM-3 (already considered). Prevents sprawl.
R-5 Should pack version be tied to MDS weight version? Yes. Weights are baked into the pack (mds_weights.yaml). Recalibrate → pack bump → eval gate. Ensures HIBT replay fidelity.

6-Dimension DQ Score
Source: 04-f4-data-quality-engine.md + 03-data-quality-spec.md + GOV-005
Formula
DQ = 0.25·freshness + 0.20·lineage + 0.20·override_accuracy + 0.15·consistency + 0.10·replay + 0.10·signal_coverage
freshness (25%)Age of each input feed vs. configured staleness threshold per source
lineage (20%)% of forecast value traceable to a versioned, hash-locked source
override_accuracy (20%)Analyst's rolling MAPE on prior overrides for this nameplate/segment
consistency (15%)Cross-surface agreement: PBI = Excel = Glean = Workbench (all surfaces same value)
replay (10%)HIBT replay match rate vs. original run; determinism target ≥ 98%
signal_coverage (10%)% of known relevant signals (dealer, tariff, macro, competitive) included
≥ 70
Green · Gate pass
No restrictions
55 – 69
Yellow · Warning badge
Analyst notified; commit allowed
< 55
Red · Hard block
GovOps review required before Gate commit

MDS — Material Disagreement Score
Source: second-opinion-agent-design-doc.md § MDS calculator · algorithm + calibration
Formula — divergent_v1 · mds_weights.yaml
MDS = W1(0.35)·point_delta_norm + W2(0.30)·(1 − band_overlap) + W3(0.20)·(1 − source_overlap) + W4(0.15)·factor_count_norm
point_delta_normAbsolute point difference normalized by forecast magnitude (volume or revenue)
band_overlapJaccard overlap of P10–P90 confidence intervals between primary and SO estimate
source_overlapUnweighted Jaccard similarity of input signal sets used by primary vs. SO agent (Q-6: may become weighted)
factor_count_normNumber of additional factors cited by SO agent not present in primary, normalized to [0,1]
< 0.10
Strong agreement
Log only, no action
0.10 – 0.29
Qualified agreement
Log as "qualified agreement"
≥ 0.30
Challenge triggered
Gate 3 blocked until SME responds or analyst dismisses (L-3)
Calibration ruleAction
Quarterly: R² between predicted MDS and realized confirmation rateIf < 0.7, trigger grid-search recalibration on golden set
Recalibration outputPack version bump (e.g., divergent_v1.1) with updated mds_weights.yaml
Pack bump gateFull 5-suite eval before deployment
> 2 GOT-01/GOT-02 events same SO issue in one cycleMandatory calibration review triggered (GOV-003 § anti-abuse)
