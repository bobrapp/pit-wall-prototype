## Cross-cutting requirements

These requirements apply to **all three** v3.0 delighter features. Each feature section assumes these and only calls out feature-specific deviations.

### CC-1 · HIBT integration

Every user-visible action and every agent-invoked computation in a v3.0 feature **MUST** write a replay-prompt-ledger entry within 5 seconds (p95). Entries include:

- Actor (user ID or agent version)
- Surface (Workbench, Outlook, Teams, Excel, SharePoint, PBI, Glean, Agent Studio)
- Action type + payload
- Prompt + source + model triple (for agent actions)
- Reproducibility hash (allows replay-from-source)
- Outcome (success / failure / dismissed / routed)

Replay determinism: ≥98% of replayed actions reproduce identical outcomes modulo timestamp and presence data.

### CC-2 · Decision-rights enforcement

Every v3.0 feature **MUST** call the Decision-Rights API before allowing any write action. Roles and rights from v2.4 baseline:

| Seat | Author override | Promote POR | Challenge | Certify agent | Audit replay |
| --- | --- | --- | --- | --- | --- |
| Leadership | ✕ | ✓ | ✕ | ✕ | ✓ (view) |
| Analyst | ✓ | ✕ | ✓ (respond) | ✕ | ✓ (view) |
| SME | ✕ | ✕ | ✓ (raise) | ✕ | ✓ (own) |
| Ops / IT | ✕ | ✕ | ✕ | ✕ | ✓ (ops scope) |
| AI GovOps | ✕ | ✕ | ✕ | ✓ | ✓ (full) |

Feature-specific rights extensions are called out in each feature section.

### CC-3 · NAMA-certified eval gate template

No v3.0 feature ships to production without passing **all five** eval gate categories:

1. **Correctness** — golden-set precision + recall ≥ stated threshold
2. **Calibration** — predicted scores correlate with realized outcomes (R² ≥ 0.7)
3. **Adversarial robustness** — designed-to-fool probes correctly classified ≥ 90%
4. **Performance & cost** — p95 latency + p95 cost within stated bounds
5. **Replay determinism** — replay-from-source matches original at ≥ 98%

GovOps (Don S.) owns the gate. Promotion ticket signed by GovOps + Architect.

### CC-4 · Phased rollout template

Each v3.0 feature follows a **5-phase rollout**:

| Phase | Audience | Duration | Exit criteria |
| --- | --- | --- | --- |
| **P1 — Internal QA** | Architect + Forecasting lead + 1 SME | 2 weeks | All eval gates green on historical data |
| **P2 — Canary** | 10% of relevant seats | 1 week | No P1 regressions; GovOps drift watch clean |
| **P3 — Expanded canary** | 50% of relevant seats | 2 weeks | Pilot retro positive; SME feedback incorporated |
| **P4 — General availability** | 100% of relevant seats | — | Public launch; auto-triggers enabled |
| **P5 — Tune & calibrate** | Production | 4–8 weeks | Success metrics on target; thresholds tuned |

Feature flags govern phase transitions. Rollback to previous phase available within 60 seconds via flag toggle.

### CC-5 · Surface-consistency SLA

Where a feature surfaces in multiple persona surfaces (Workbench + Outlook + Teams + PBI), all surfaces **MUST** reflect the same value within the v2.4 baseline SLA of **2.4 seconds p95**. No regression of the existing SLA is permitted.

### CC-6 · Feature flag policy

Each feature gets a dedicated kill switch flag:

- `pw.second-opinion` — default off, enable per-seat
- `pw.ghost-lap` — default off, enable per-seat
- `pw.championship` — default off, enable globally

All three flags must be **kill-switched** within 60s by GovOps in case of incident.

### CC-7 · Cost guardrails

Every agent-invoking feature has a **per-seat daily budget** enforced at the agent layer. Defaults:

- Second-Opinion Agent: $4.00/seat/day
- Ghost Lap: $0.50/seat/day (mostly cached precomputes)
- Championship Table: $0.10/seat/day (read-only)

Alert fires at 80% utilization; hard cap at 110%.

### CC-8 · Privacy & visibility

All three features **MUST** honor seat-level visibility:

- SMEs see their own challenges and contributions; do not see other SMEs' raw scores
- Leadership sees aggregate views and decision-rights-permitted detail
- GovOps sees full replay-lineage including all actors
- Ops sees pipeline + agent-system data; not analyst content

## Feature 1 · Second-Opinion Agent

**Feature flag:** `pw.second-opinion`
**Owner:** Bob Rapp (Architect) + Don S. (GovOps)
**Status:** Spec — pending sponsor signoff
**Target GA:** v3.0, end of Q3 2026

### 1.1 · Problem statement

In v2.4, every override and approval is grounded in a single agent's analysis (workbench-anomaly + narrative-drafter) plus the human reviewer's judgment. This produces two failure modes:

1. **Anchored optimism** — when the model recommends, the human tends to ratify rather than independently challenge
2. **Single-source blind spots** — model uses one prompt template, one source weighting, one seed; systematic biases go undetected

We need a deliberately adversarial second pass before any plan-of-record commit on material forecast sets.

### 1.2 · Vision

One click on any override draft or approval card invokes a **second NAMA-certified agent** with a different prompt template, different source weighting, and different model temperature. The second opinion is compared to the primary analysis. Material disagreements (score ≥ 0.3) raise a **structured challenge** with reason code SO-XX and route to the appropriate SME.

Not a vote. Not a tiebreaker. A surfacer of disagreement.

### 1.3 · User stories

- **As an Analyst**, I want to invoke a second-opinion review before I stage my override, so I know whether a different framing would produce a different number.
- **As a Leader**, I want every plan-of-record commit on a forecast set above the materiality threshold to have passed second-opinion review, so my approvals are robust.
- **As an SME**, I want to receive structured second-opinion disagreements with full source overlap and divergent factors, so I'm not pulled in for agent-versus-agent noise.
- **As GovOps**, I want every second-opinion run logged and replayable, so I can audit pattern of agreement/disagreement over time.

### 1.4 · Functional requirements

- **FR-SO-1** — One-click *Get second opinion* button on every override draft card (Analyst seat), every approval card (Leadership seat), and every challenge resolution card (SME seat).
- **FR-SO-2** — Auto-trigger on Gate 3 (plan-of-record commit) for any forecast set with revenue impact ≥ $50M or volume impact ≥ 5k units. Threshold configurable per nameplate.
- **FR-SO-3** — Second agent uses **deliberately divergent prompt pack**: different framing (e.g., bear vs. bull narrative seed), different source weighting (Avista weight 1.5x, internal weight 0.7x), different temperature (+0.2 vs primary).
- **FR-SO-4** — Compute **material-disagreement score** (MDS) on a 0–1 scale based on: point-estimate delta, confidence-band overlap, source-evidence overlap, divergent-factor count.
- **FR-SO-5** — MDS ≥ 0.3 raises a structured challenge object with reason code SO-XX (see § 1.10). MDS < 0.3 logs a clean second-opinion confirmation; no challenge raised.
- **FR-SO-6** — Disagreement report includes (in order): primary recommendation, second-opinion recommendation, delta, MDS components, source-overlap %, top 3 divergent factors, suggested SME routing.
- **FR-SO-7** — Auto-route raised challenge to SME based on top divergent factor:
  - Incentive elasticity → Pricing team
  - Market / dealer signal → MIA (Jim Kyle's team)
  - Inventory / production constraint → Ops + supply chain
  - Tariff / regulatory → External-affairs
  - Competitive launch → MIA
- **FR-SO-8** — Owner can dismiss with reason code (DISM-1 spurious, DISM-2 stale, DISM-3 already-considered) or open as full challenge. Dismissals logged.
- **FR-SO-9** — Every run writes a HIBT entry with full reproducibility hash; replay-from-source must reproduce identical recommendation modulo timestamp.
- **FR-SO-10** — Surfaces in: Workbench (analyst), approval card (Leadership), Teams (SME on routing).

### 1.5 · Non-functional requirements

- **NFR-SO-1** — Latency: p95 second-opinion completion ≤ 90 seconds end-to-end.
- **NFR-SO-2** — Cost: average ≤ $0.20/run in production; p95 ≤ $0.40/run; per-seat daily cap $4.00 (CC-7).
- **NFR-SO-3** — Throughput: support 50 concurrent runs without queue depth > 5 (peak: monthly cycle commit window).
- **NFR-SO-4** — Reliability: 99.5% successful completion; failures degrade gracefully ("Second opinion unavailable, proceed with single-source analysis").
- **NFR-SO-5** — Replay determinism per CC-1 (≥98%).

### 1.6 · Acceptance criteria

- **AC-SO-1** — Manual invocation: user clicks *Get second opinion*; full disagreement report renders in the side panel within 90s p95.
- **AC-SO-2** — Auto-trigger: every Gate 3 commit on forecast sets ≥ $50M impact has an attached second-opinion run logged in HIBT.
- **AC-SO-3** — MDS calibration: realized SME-confirmation rate on raised challenges ≥ 70%.
- **AC-SO-4** — False-positive rate: ≤ 15% of raised challenges dismissed by SME as "spurious" (DISM-1).
- **AC-SO-5** — HIBT replay: 50 replays sampled randomly from production; ≥ 98% reproduce identical recommendation.
- **AC-SO-6** — Cost: 7-day rolling average per-run cost ≤ $0.20.
- **AC-SO-7** — Failure mode: when second-opinion service is unavailable, the override / approval flow degrades gracefully — never blocks the primary path.

### 1.7 · Eval gates (NAMA-certified)

Must pass all five before any phase transition above P1.

| Gate | Eval | Threshold |
| --- | --- | --- |
| **Correctness** | 100-case golden set, SME-judged disagreements | ≥ 85% precision, ≥ 80% recall |
| **Calibration** | MDS predicted vs. SME-confirmation realized | R² ≥ 0.7 |
| **Adversarial** | 20 designed-to-fool probes (close-but-different, similar-looking signal) | ≥ 90% correct |
| **Perf + cost** | 100 runs at peak traffic shape | p95 latency ≤ 90s; p95 cost ≤ $0.30 |
| **Replay determinism** | 50 random replays | ≥ 98% reproduce identical |

Gate sign-off: Don S. (GovOps).

### 1.8 · Phased rollout (per CC-4)

- **P1 — Internal QA (weeks 1–3)** — Krisztina + Jim + Bob test against Q4 2026 historical data. Exit: all five eval gates green.
- **P2 — Canary 10% (week 4)** — Analyst seats only; manual invocation only (no auto-trigger). Exit: SME-confirmation ≥ 70%, no incidents.
- **P3 — Expanded canary 50% (weeks 5–6)** — All Analyst + Leadership seats; manual + auto-trigger on forecast sets ≥ $100M. Exit: pilot retro positive, GovOps drift clean.
- **P4 — GA (week 7)** — 100% seats; auto-trigger at $50M threshold. SME routing live.
- **P5 — Tune (weeks 8–12)** — Calibrate MDS thresholds based on production SME-confirmation rates by domain.

### 1.9 · Dependencies

- **HIBT v2.0** — for replay-prompt-ledger (CRITICAL — gates GA)
- **workbench-anomaly v3.5** — provides source-weighting hooks (LIVE in v2.4)
- **glean-copilot v2.8** — alternate retrieval path for divergent sourcing (in flight)
- **narrative-drafter v2.4** — for second-opinion report drafting (in flight)
- **Decision-Rights API** — for SME routing authorization (LIVE)

### 1.10 · Risks & mitigations

| ID | Risk | Mitigation |
| --- | --- | --- |
| R-SO-1 | Two AIs agreeing creates false confidence in the human reviewer | Surface source-overlap explicitly; require human signoff regardless; "agreement is not validation" copy throughout |
| R-SO-2 | 90-second latency makes feature feel optional; analysts skip it | Pre-warm during Pit Stop 2 (agent prep); background-run during analyst review so result is ready by Pit Stop 3 |
| R-SO-3 | Cost spike if promiscuously invoked | Per-seat daily budget (CC-7); alert at 80%; hard cap at 110% |
| R-SO-4 | SME-routing miscategorizes divergent factor; wrong person gets pulled in | Owner can re-route with reason code; routing classifier eval gate ≥ 90% |
| R-SO-5 | Divergent prompt pack drifts from primary over time; comparison becomes apples-to-oranges | Quarterly recalibration; divergence-from-primary monitored as a drift signal |

### 1.11 · Reason codes (new)

- **SO-12** Incentive-elasticity disagreement
- **SO-13** Source-evidence overlap < 30%
- **SO-14** Confidence-band non-overlap
- **SO-18** Competitive-launch reframe
- **SO-21** Dealer-pulse divergence
- **DISM-1** Dismissed — spurious
- **DISM-2** Dismissed — stale signal
- **DISM-3** Dismissed — already considered

## Feature 2 · Ghost Lap

**Feature flag:** `pw.ghost-lap`
**Owner:** Bob Rapp (Architect) + Krisztina Gilezan (Forecasting)
**Status:** Spec — pending sponsor signoff
**Target GA:** v3.0, end of Q3 2026

### 2.1 · Problem statement

In v2.4, the Pit Wall is excellent at "what is the forecast right now?" but weak at "how did we get here, and did we improve?" Three failure modes today:

1. **Retrospective amnesia** — analysts don't have an easy way to compare this cycle's forecast to last cycle's at the same point in time, so accuracy lessons don't compound
2. **Recurring-challenge blindness** — same SME, same reason code, same nameplate raises a challenge cycle after cycle; the wall doesn't surface the pattern
3. **Override-outcome trail loss** — when an analyst overrides today, they can't see what happened to similar overrides last cycle (was it correct? overruled? cycled?)

The ghost lap solves all three with a single temporal overlay.

### 2.2 · Vision

A **toggleable overlay** on every quantile fan chart shows the prior cycle's forecast (and actuals, where elapsed) as a semi-transparent "ghost" layer. Side-by-side accuracy reports surface at Gate 2 (challenge-ready). Recurring challenges are explicitly tagged so the system stops asking the SME the same question twice.

### 2.3 · User stories

- **As an Analyst**, I want to see how my prior cycle's forecast compared to actuals, so I learn what I got right and wrong.
- **As an SME**, I want to see if a challenge I raised last cycle was resolved, so I don't waste time re-raising it.
- **As a Leader**, I want a "did we improve" view at S&OP, so retrospective is data-driven rather than vibes-based.
- **As GovOps**, I want recurring-challenge patterns surfaced as system signals, so I can route them into prompt or memory improvements.

### 2.4 · Functional requirements

- **FR-GL-1** — Toggle *Ghost Lap* on any forecast Workbench chart; overlays prior cycle's data as a semi-transparent layer (default 35% opacity, user-adjustable).
- **FR-GL-2** — Ghost layer data includes: prior baseline (P50), prior consensus (overridden line), prior actual (where elapsed), prior P10/P90 band, prior committed plan-of-record.
- **FR-GL-3** — **Accuracy delta report**: side-by-side `|this-cycle forecast at month T - actual at T|` vs. `|last-cycle forecast at T - actual at T|`. Surfaced as a strip under the chart.
- **FR-GL-4** — **Recurring-challenge detection**: if the same `(SME × reason code × nameplate)` triple raises a challenge in N consecutive cycles (default N=3), tag as `ghost-issue` and surface in the challenge inbox with an amber recurring badge.
- **FR-GL-5** — **Override-outcome trail**: for each override staged in current cycle, show what happened to similar overrides last cycle. Similarity = `(nameplate × reason-code × sign-of-delta)`. Outcome categories: correct, overruled, cycled, dismissed.
- **FR-GL-6** — **Gate-2 auto-report**: at Gate 2 (challenge-ready), system auto-generates a ghost-lap diff report and attaches to the forecast set. Available to all seats per decision-rights.
- **FR-GL-7** — **Exec brief embed**: Leadership can optionally include a 1-page ghost-lap summary in the monthly exec brief (off by default; opt-in per cycle).
- **FR-GL-8** — **Surface ghost-issue at challenge raise time**: when an SME starts a new challenge, the form checks for matching `(SME × reason-code × nameplate)` history and warns "You raised this challenge in N of the last M cycles — it has N=K open and N=L resolved status".
- **FR-GL-9** — Surfaces in: Workbench charts, exec brief (opt-in), Teams (challenge inbox), Power BI (Leadership tile).

### 2.5 · Non-functional requirements

- **NFR-GL-1** — Ghost-lap data renderable in < 2 seconds p95 for any forecast set. Achieved via nightly precompute.
- **NFR-GL-2** — Storage: 24 cycles of full-lineage history retained per forecast set; older compressed to summary stats only.
- **NFR-GL-3** — Privacy: honors decision-rights model; SMEs see their own historical challenges, not other SMEs'.
- **NFR-GL-4** — Recurring-challenge detection runs nightly; tag updates available by 06:00 ET each weekday.

### 2.6 · Acceptance criteria

- **AC-GL-1** — Toggling ghost lap on a chart renders prior cycle within 2s p95 (warm cache) and within 4s p95 (cold cache).
- **AC-GL-2** — Recurring-challenge classifier: ≤ 5% false-positive rate on 50-case golden set; N threshold tunable per nameplate.
- **AC-GL-3** — Override-outcome trail similarity scoring: ≥ 0.85 agreement with human-labeled pairs.
- **AC-GL-4** — Gate-2 diff report auto-generated for 100% of cycles; never requires manual intervention.
- **AC-GL-5** — Privacy: 20 cross-SME test cases (SME A trying to view SME B's challenges); 0 leakage events.
- **AC-GL-6** — Visual: ghost layer renders within design spec on all 5 chart variants (quantile fan, scenario compare, override stack, anomaly overlay, recurring-issue annotation).
- **AC-GL-7** — When N=0 historical cycles exist (new forecast set), feature degrades to "no ghost available" message — does not error.

### 2.7 · Eval gates (NAMA-certified)

| Gate | Eval | Threshold |
| --- | --- | --- |
| **Correctness** | 50-case golden set for recurring-challenge classifier | ≥ 95% precision @ ≥ 80% recall |
| **Calibration** | Override-similarity scorer agreement with human labels | ≥ 0.85 |
| **Adversarial** | 15 confounder cases (near-duplicate reason codes, swapped sign, ambiguous nameplate) | ≥ 90% correct |
| **Perf + cost** | 100 chart-toggle interactions | p95 ≤ 2s (warm); cost < $0.05/toggle |
| **Privacy / replay** | 20 cross-SME visibility probes + 30 replay tests | 0 leakage; ≥ 98% replay reproduce |

Gate sign-off: Don S. (GovOps).

### 2.8 · Phased rollout (per CC-4)

- **P1 — Internal QA (weeks 1–2)** — Build read-only ghost-lap view; backtest against historical Q1–Q3 2026 cycles. Krisztina + Jim QA.
- **P2 — Canary 10% (week 3)** — Pilot with Krisztina + 2 analysts on real Q4 cycle. Manual toggle only; no Gate 2 auto-report yet.

## Feature 3 · Championship Table

**Feature flag:** `pw.championship`
**Owner:** Bob Rapp (Architect) + Ramzi Abdelmoula (Sponsor — communications)
**Status:** Spec — pending sponsor signoff
**Target GA:** v3.0, end of Q3 2026

### 3.1 · Problem statement

In v2.4 we measure the substrate (forecast accuracy, drift, SLA), but we don't celebrate the **engineers and SMEs** whose judgment makes the substrate better. Two failure modes:

1. **Invisible excellence** — the analyst whose overrides consistently lift MAPE by +1.5 pts is invisible; the SME whose challenges materially improve the plan is invisible
2. **No improvement flywheel** — there's no light-touch positive reinforcement loop for the behaviors we want to see (high-signal challenges, evidence-attached overrides, agent improvement contributions)

Note the explicit **anti-goal**: this is **not** a performance review tool. It must never replace 1:1 conversations or drive comp decisions. If it ever feels punitive, it has failed.

### 3.2 · Vision

A quarterly **Grand Prix** with monthly micro-seasons across four leagues. Recognizes contributions, not failures. Opt-in visibility for individuals; aggregate views always available. Tied to S&OP cadence so podium recognition happens at the right organizational moment.

### 3.3 · User stories

- **As an Analyst**, I want to see how my overrides compared to actuals over time, so I learn and improve.
- **As an SME**, I want recognition when my challenges meaningfully improve the plan.
- **As a Leader**, I want to celebrate engineers who consistently strengthen the forecast.
- **As GovOps**, I want to celebrate agent improvements that materially lift accuracy or reduce rework.
- **As anyone**, I want to give a colleague a "lap clap" when I see them do good work — a low-friction recognition gesture.

### 3.4 · Functional requirements

- **FR-CT-1** — Championship Table page at `/championship` in the Pit Wall navigation; visible to all seats per decision-rights.
- **FR-CT-2** — **Four leagues** per season:
  1. **Override Value-Add** (Analyst seats)
  2. **Challenge Precision** (SME seats)
  3. **Agent Improvement** (GovOps + Analyst seats)
  4. **Cross-League Mentorship** (anyone, peer-nominated)
- **FR-CT-3** — **Seasons**: monthly micro-seasons + quarterly Grand Prix. Tied to S&OP cadence — micro-season closes at Gate 3 (POR commit) each cycle; Grand Prix closes at quarter end.
- **FR-CT-4** — **Scoring metrics**:
  - **Override Value-Add**: Σ `(MAPE_no_override − MAPE_with_override) × $impact`, weighted by recency (more recent = higher weight, half-life 90 days)
  - **Challenge Precision**: `(challenges raised that became material POR changes) / (total challenges raised)`, with minimum 5-challenge floor to qualify
  - **Agent Improvement**: Σ `(eval-score delta × adoption-rate × $impact)` for agent versions promoted in season
  - **Mentorship**: peer-nominated; recognizes mentees crediting a mentor in their own scoring period. Capped at 3 nominations per quarter per mentor.
- **FR-CT-5** — **Visibility controls**: every user opts in to public ranking via a one-time settings choice (default = aggregate-only). Aggregate position shown as quartile band, never raw number.
- **FR-CT-6** — **Quarterly Podium ceremony**: 1st / 2nd / 3rd in each league surfaced to Leadership at quarterly S&OP. Leadership decides recognition format (call-out, gift, etc.) — system surfaces, humans celebrate.
- **FR-CT-7** — **Anti-gaming guardrails**:
  - Minimum 5 contributions per season to qualify
  - Outliers (top 1%) reviewed by GovOps before recognition
  - Score formula reproducible from HIBT data; replay-from-source produces identical leaderboard
  - No score appears in performance review surfaces or comp tools (technical guarantee + policy)
- **FR-CT-8** — **Lap clap**: any user can give a "lap clap" to a peer's specific contribution (an override, a challenge, an agent improvement). Capped at 5 claps given per user per season; max 20 received per user per season. Counts as a small bonus in scoring (≤ 5% of total possible score).
- **FR-CT-9** — **Improvement-over-time view**: every user can view their own scoring trend over the last 4 quarters, regardless of public ranking opt-in.
- **FR-CT-10** — Surfaces in: Workbench (`/championship`), Workbench in-context recognition badges (small, non-intrusive), exec brief Q-end addendum (optional, Leadership opt-in).

### 3.5 · Non-functional requirements

- **NFR-CT-1** — Refresh latency: leaderboard updates within 5 seconds of any HIBT event affecting a score.
- **NFR-CT-2** — Storage: 2 years of championship history retained (8 quarterly seasons + monthlies).
- **NFR-CT-3** — Privacy: never expose individual scores below the median band to other users. Always expose own score to self.
- **NFR-CT-4** — Performance: 500 concurrent users viewing the table — p95 ≤ 2s.
- **NFR-CT-5** — Auditability: every score reproducible from HIBT.

### 3.6 · Acceptance criteria

- **AC-CT-1** — Page renders all four leagues with current season standings; respects opt-in visibility settings.
- **AC-CT-2** — Scoring formula reproducible from HIBT data; 100 random replays produce identical leaderboard.
- **AC-CT-3** — Opt-out users do not appear by name anywhere; aggregate quartile band is preserved.
- **AC-CT-4** — Anti-gaming: 100% of submissions below 5-contribution floor are correctly excluded.
- **AC-CT-5** — Lap-clap caps enforced: max 5 given/season, max 20 received/season; system blocks attempts beyond cap.
- **AC-CT-6** — Quarterly Podium auto-generated within 24h of Grand Prix close (Q-end S&OP).
- **AC-CT-7** — User can always view own improvement-over-time, regardless of public opt-in.
- **AC-CT-8** — Zero data exposed to comp / HR systems (verified via integration audit).

### 3.10 · Risks & mitigations

| ID | Risk | Mitigation |
| --- | --- | --- |
| R-CT-1 | Public ranking feels like performance review (anti-goal) | Default opt-out for public visibility; quartile band display only; Leadership messaging at launch "this is for celebration, not eval"; never tied to comp |
| R-CT-2 | Gaming behavior — stage easy overrides to inflate score | Minimum-contribution floor; impact-weighted scoring; outlier review by GovOps; eval gate 100% gaming detection |
| R-CT-3 | Demotivating for engineers consistently lower-quartile | Mentorship league as an alternate path to recognition; improvement-over-time view always available to self; opt-out always available |
| R-CT-4 | Becomes the goal instead of the byproduct (perverse incentive) | Leadership briefed never to reference table during 1:1s; recognition tied to celebration moments only (Podium ceremony); cultural sign-off from Ramzi required at GA |
| R-CT-5 | Lap-clap becomes a popularity contest | Per-user clap caps; capped contribution to score (≤ 5%); GovOps reviews recipients who exceed 15 claps received |
| R-CT-6 | First Grand Prix produces unexpected unfair-feeling winners | Hold private Podium retro between Krisztina + Ramzi + Bob before public ceremony; allow score adjustments with documented reason |

## Cross-feature dependencies & sequencing

All three features depend on **HIBT v2.0** general availability. HIBT v2.0 is the gating dependency for the entire v3.0 release.

HIBT v2.0 GA  →  ┬→  Second-Opinion Agent  ─→  GA week 7

                 └→  Championship Table   ─→  GA week 5
