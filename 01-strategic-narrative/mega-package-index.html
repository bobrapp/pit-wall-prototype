02 · Architecture decisions

Five locked decisions.

ADRs in Michael Nygard format. Each one names the alternative we rejected and why. These are the architectural guardrails the system will live by.

ADR-001

One trusted substrate

All writes through one Layer A. Rejects federated and SharePoint-of-truth alternatives.

docs/adr/ADR-001-…md
ADR-002

Five-seat persona model

Fixed 5 seats with API-enforced decision rights. Rejects free-form roles and super-user.

docs/adr/ADR-002-…md
ADR-003

HIBT replay-prompt-ledger

≥98% replay determinism as hard NFR. Append-only; corrections by reference.

docs/adr/ADR-003-…md
ADR-004

NAMA-certified agent gate

5-category eval gate. GovOps owns. Rejects shadow-mode-only as permanent state.

docs/adr/ADR-004-…md
ADR-005

Dual cadence (15-min + monthly)

Two clocks, one substrate. Calls out mid-cycle overlap as highest-risk implementation area.

docs/adr/ADR-005-…md
